Local‑First Expense Tracker — Concept Reference

A concise single‑file reference: branding, features, UX, tech stack, sync approach, schema, and roadmap — saved for future work.

Branding

Name ideas: Ledge, Minty, Pondo, PesoPad, CacheFlow, Tab, Kurso

Tone: Simple · Stable · Privacy‑first

Heading: Poppins / Body: Inter

Core Concept

  • App works 100% offline — SQLite is the primary local store.
  • Syncs automatically when online with conflict resolution (last‑write or CRDT for advanced).
  • Privacy‑first: local encryption, optional cloud backup.
  • Target: people with intermittent internet and small business owners.

Features

MVP

  • Local add/edit/delete transactions (amount, category, wallet, notes).
  • Daily / monthly views and simple analytics.
  • Background sync + queued ops.
  • PIN / biometric lock and encrypted local keys.

Phase 2

  • Recurring transactions, budgets, receipt photos, CSV export, multiple wallets.

Future / Power

  • AI categorization, shared wallets, subscription parsing, CRDT full sync.

UX Highlights

Simple flows, fast quick‑add, minimal fields.

  • Home: today total, quick add, recent tx.
  • Add: amount (numpad) → category → wallet → save.
  • Analytics: pie (category), trend (monthly), largest spends.

Security

  • Encrypt sensitive data, use SecureStore for keys.
  • App lock (PIN / biometrics).

Tech Stack

Layer Recommendation
Frontend React Native (Expo), Zustand or Jotai for state
Local DB SQLite (expo-sqlite) + Drizzle ORM (optional)
Networking React Query or custom sync engine, queueing offline ops
Backend Supabase (fast) or custom Express + Postgres (control)
Storage/Backup Supabase Storage / encrypted JSON exports
Auth Supabase Auth or email+PIN locally (optional)

Sync Approach (Simple)

Flow 
1. Client records tx locally with updatedAt timestamp and a UUID.
2. When online, client sends all local records changed since lastSync to API.
3. Server merges (last-write-wins by timestamp) and returns authoritative rows.
4. Client applies server rows to local DB and marks synced.

Advanced: use CRDTs (e.g., Automerge) or vector clocks for stronger conflict handling.

Database Schema (example)

Table Columns (key)

Developer Notes & Tips

  • Keep local DB as the source of truth — UI reads directly from it.
  • Implement an idempotent sync API so retries are safe.
  • Use small JSON diffs for sync payloads (send only changed rows).
  • Test offline → online heavily on real devices (Philippines network patterns included).
